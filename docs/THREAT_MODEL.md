作成日: 2026-03-21

根拠: ADR 設計提案（35 ADR）、REQUIREMENTS.md、OpenClaw 教訓マトリクス、OWASP LLM Top 10 (2025)、学術論文

## 1. 脅威モデルの目的

Clawless が保護する資産、想定する攻撃者、攻撃面、各脅威への対策マッピングを定義する。REQUIREMENTS.md（FR/NFR）と ADR 設計提案を根拠とし、「何を・誰から・どう守るのか」を体系的に整理する。

## 4. 攻撃面（Attack Surface）

### 4.1 ツール実行面

Claude Code は以下のツールを通じてホストシステムと対話する。各ツールが攻撃面となる。

| ツール   | リスク | 攻撃例                                               |
| -------- | ------ | ---------------------------------------------------- |
| Bash     | 最大   | 任意コマンド実行、ファイル操作、ネットワーク通信     |
| Edit     | 高     | 設定ファイル改竄、バックドア挿入                     |
| Write    | 高     | 新規ファイル作成による設定上書き                     |
| Read     | 中     | 機密ファイル読取                                     |
| MCP      | 高     | MCP サーバー経由の任意操作、Elicitation フィッシング |
| WebFetch | 中     | 外部 URL へのデータ送信（クエリパラメータ埋め込み）  |

### 4.2 入力面

| 入力経路             | リスク | 攻撃例                                               |
| -------------------- | ------ | ---------------------------------------------------- |
| ユーザープロンプト   | 中     | 意図しないプロンプトインジェクション                 |
| ファイルコンテンツ   | 高     | 読み込んだファイルに埋め込まれた間接インジェクション |
| MCP Elicitation      | 中     | フィッシング URL を含む構造化入力要求                |
| チャンネルメッセージ | 高     | 外部メッセージ経由のインジェクション                 |
| 依存パッケージ       | 高     | install 時のポストインストールスクリプト             |

### 4.3 Windows ネイティブ固有の攻撃面

| 攻撃面                | 説明                                                   | 他 OS との差異          |
| --------------------- | ------------------------------------------------------ | ----------------------- |
| OS サンドボックス不在 | Defense Layer 3 が利用不可                             | Linux/WSL2 では自動有効 |
| NTFS ADS              | 代替データストリーム（file.txt:$DATA）によるデータ隠蔽 | NTFS 固有               |
| NTFS Junction         | ジャンクションポイントによるパス迂回                   | NTFS 固有               |
| UNC パス              | \\\\server\\share による外部ファイルアクセス           | Windows 固有            |
| 8.3 短縮名            | C:\\PROGRA~1 による deny パターン回避                  | Windows 固有            |
| cmd.exe / PowerShell  | シェルフォールバック、メタキャラクタの差異             | Windows 固有            |
| .cmd/.bat cwd         | 作業ディレクトリインジェクション                       | CVE-2026-31999          |

## 6. リスク評価マトリクス

### 6.1 評価基準

影響度: Critical（認証情報漏洩）/ High（データ漏洩）/ Medium（設定改竄）/ Low（情報収集）

発生可能性: Very Likely（既知の攻撃、実証済み）/ Likely（理論的に可能、一部実証）/ Unlikely（高度な専門知識が必要）/ Rare（極めて限定的な条件下）

## 7. 防御の限界（正直な記載）

### 7.1 フックベース防御の構造的限界

| 防げること                                               | 防げないこと                                           |
| -------------------------------------------------------- | ------------------------------------------------------ |
| 既知のツールスイッチングパターン（sed -i, python -c 等） | 未知のインタプリタ・言語での間接実行                   |
| 既知のパス難読化（/proc/self/root, 8.3 短縮名）          | 新発見のパス解決バグ                                   |
| フック設定ファイルの直接変更                             | permissions.deny をバイパスする未知の Claude Code バグ |
| ゼロ幅文字・ホモグリフ                                   | 将来発見される新しいエンコーディング攻撃               |
| deny パターン内のコマンド                                | Windows ネイティブでの子プロセスの OS レベル制御       |

### 7.2 Windows ネイティブ固有の限界

OS サンドボックス（Defense Layer 3）が利用不可のため、以下のリスクが残存する。

| 残存リスク                   | 説明                                     | 緩和策                                            | OpenShell 緩和（Layer 3b, ADR-037） |
| ---------------------------- | ---------------------------------------- | ------------------------------------------------- | ----------------------------------- |
| 子プロセスのファイルアクセス | パイプチェーン・難読化された間接アクセス | PreToolUse のパターンマッチで大半を検出           | Landlock LSM denyWrite / denyRead   |
| raw ソケット通信             | Bash 内から直接ソケットを開く操作        | エンタープライズ: Windows Firewall 送信規則を推奨 | Seccomp BPF socket deny             |
| DNS トンネリング             | DNS クエリによるデータ流出               | sandbox.network（WSL2 / Linux のみ）              | Network Namespace deny_all_other    |
| PowerShell ソケット          | [System.Net.Sockets.TcpClient] 等        | パターンマッチで既知パターンを検出                | Seccomp BPF + Network Namespace     |

設計原則: フックで主要攻撃ベクトルの 95% をカバーし、残り 5% のリスクを本文書で明文化する（NFR-02-03）。OpenShell（Layer 3b）はオプション強化として残存リスクを低減する（ADR-037）。

### 7.3 継続的な脅威への対応

未知の攻撃は以下の仕組みで早期検知・対応する。

| メカニズム     | ADR     | 説明                                                |
| -------------- | ------- | --------------------------------------------------- |
| 証跡記録       | ADR-013 | SHA-256 hash chain で全操作を記録。事後分析が可能   |
| 意図検出フック | ADR-027 | agent タイプフックで推論過程を検査                  |
| パターン更新   | ADR-029 | injection-patterns.json の定期更新（30 日以内推奨） |
| バージョン追従 | ADR-029 | Claude Code の新機能・セキュリティ修正を自動検知    |
| 学術研究の導入 | ADR-027 | Tool-Input/Output Firewall, MELON, CoT 監査         |
