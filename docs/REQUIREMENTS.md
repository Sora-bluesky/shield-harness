# Clawless 要件定義書

作成日: 2026-03-21
階層: Why（戦略）
根拠: ADR 設計提案（35 ADR）、OpenClaw 教訓マトリクス（5 カテゴリ 18 問題）、HANDOFF.md

---

## 1. プロジェクト概要

### 1.1 名称と由来

Clawless（くろうレス = 苦労レス = 苦労しない）。Claude Code が安全・再現可能に開発を進めるためのハーネス。`.claude/` ディレクトリに展開される hooks + rules + skills + settings.json の総体として定義される。

### 1.2 解決すべき課題

OpenClaw は「メッセージアプリから 24/7 AI エージェントを操作する」という革新的な体験を実現した。しかし運用の中で 5 カテゴリ 18 件のセキュリティ問題（うち 11 件に CVE 付与）が発生し、「便利だが危険」という状態に陥った。

Clawless は OpenClaw の本質的価値（M-1〜M-5）を継承しつつ、18 件の問題すべてをフックベースの強制メカニズムで構造的に根絶する。「CLAUDE.md に書いたルールが守られない」という根本課題を、「ルール違反を構造的に不可能にする」アーキテクチャで解決する。

### 1.3 ステークホルダー

| ステークホルダー         | 関心事                                                                                         |
| ------------------------ | ---------------------------------------------------------------------------------------------- |
| 個人開発者（プライマリ） | VS Code から Claude Code を安全に使いたい。設定は苦労なく、セキュリティは自動で効いてほしい    |
| エンタープライズ管理者   | 組織のセキュリティポリシーを開発者に強制したい。コスト制御・データ境界・監査証跡が必要         |
| OSS コントリビュータ     | フックやパターンを追加・改善したい。拡張ポイントが明確であってほしい                           |
| Claude Code 本体         | Clawless のフック群から指示を受ける側。ネイティブ API（hooks / permissions / sandbox）の利用者 |

### 1.4 スコープ

| 項目                   | スコープ内                                    | スコープ外                      |
| ---------------------- | --------------------------------------------- | ------------------------------- |
| OS                     | Windows ネイティブ（プライマリ）、WSL2、Linux | WSL1、macOS（将来対応候補）     |
| インターフェース       | VS Code Claude Code 拡張パネル                | 独立ターミナル、Web UI          |
| Claude Code バージョン | v2.1.45 以降（30+ フックイベント対応）        | v2.1.44 以前                    |
| チャンネル             | Phase 1: OpenClaw 方式、Phase 2: --channels   | 独自メッセージングプロトコル    |
| セキュリティ           | アプリケーション層防御（permissions + hooks） | OS カーネルレベル隔離の独自実装 |

## 4. 制約条件

### 4.1 技術的制約

| ID   | 制約                                                                | 影響                                                       | 対応方針                                                                      |
| ---- | ------------------------------------------------------------------- | ---------------------------------------------------------- | ----------------------------------------------------------------------------- |
| C-01 | Windows ネイティブでは OS レベルサンドボックスが未対応（"planned"） | sandbox.filesystem, sandbox.network が機能しない           | Defense Layer 1-2（permissions + hooks）で 95% を防御。残り 5% はリスク文書化 |

| C-05 | Claude Code の permissions.deny はパターンマッチ（glob）ベース      | 難読化されたコマンドのすべてをパターンで捕捉するのは不可能 | PreToolUse フックでパス正規化 + NFKC 正規化を行い、正規化後に照合             |

## 5. OpenClaw 教訓の要件化

## 6. 優先度定義

| 優先度     | 定義                                                                                 | 対象                                                   |
| ---------- | ------------------------------------------------------------------------------------ | ------------------------------------------------------ |
| **Must**   | Clawless として最低限動作するために必須。欠落すると OpenClaw の問題が再発する        | FR-01〜FR-08 の Must 項目、NFR-01〜NFR-02              |

| **Could**  | 将来の拡張候補。本要件定義には含めない                                               | Mac 対応検証、GUI ダッシュボード、クラウドホスティング |

## 7. 受入基準（全体）

Clawless が「完成」と判定される条件:

1. `npx clawless init --profile standard` が Windows ネイティブ環境で正常に完了する
2. 22 フックスクリプト全てが settings.json に登録され、テスト入力で正常応答する
3. OpenClaw 18 問題に対応する攻撃シミュレーションが全て deny される
4. 7 種のフック回避攻撃が全て検出・ブロックされる
5. evidence-ledger.jsonl の hash chain が改竄検知テストに合格する
6. CLAUDE.md 28 ルールの全てにフック強制が存在し、マッピング表で確認できる
7. `npx clawless health` がエラーなしで完了する
